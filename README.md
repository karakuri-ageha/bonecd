### 設計思想

| 項目 | 内容 |
|---|---|
| コンセプト | 身体情報を軸に、キャラクターの表層から深層まで全要素を体系的に記録・整理するモジュール式フレームワーク |
| 設計原則 | 1. 身体を軸に据える<br>2. モジュール式の構成を採用する<br>3. ジャンルによる分類を廃止する<br>4. レーティングによるレベル分けを廃止する<br>5. 性別パーツはカテゴリ別に選択する<br>6. フォーマット全体がプロフィールである<br>7. キャラクターシートの範疇を明確にする<br>8. 内在/外在で情報の所在を区別する<br>9. 共通/拡張で情報の粒度を区別する<br>10. 二つの使われ方を想定する |
| 禁忌事項 | なし。BONECDは器であり、使い方に禁忌はない。 |
| 期待効果 | ・キャラクター情報の一元管理<br>・チーム間でのキャラ設定共有の効率化<br>・設定の抜け漏れの発見<br>・キャラクターの深掘りによる解像度向上<br>・ジャンルや性別に縛られない自由なキャラクター設計 |

### 推奨活用法

| 目的 | 推奨ルート |
|---|---|
| まず試したい | 第3章「使用方法」の用途別おすすめ構成を確認し、共通ベースだけで1キャラ作ってみる |
| 全体像を把握したい | 第1章「概要」→ 第2章「全パーツ一覧」の順に読む |
| 使い方を知りたい | 第3章「使用方法」でフロー・選択基準・差し込み位置を確認する |
| パーツの中身を確認したい | 第4〜7章で該当パーツのテーブル・説明・記入例を参照する |
| 完成形のイメージを掴みたい | 付録B「記入サンプル」で共通ベース21パーツの記入済み例を見る |
| よくある疑問を解消したい | 付録A「FAQ」（全44問）から逆引きする |
| 深掘りしたい | 必要を感じた拡張パーツだけ第5章・第7章から選んで追加する |
| 自分用に整えたい | 不要な空欄・パーツを削除して自分仕様にする。第3章の記入ガイドライン参照 |
| アプリで試したい | 付録A「FAQ」のQ37を確認してからプロトタイプアプリを使ってみる |
