## 1. 概要

### システム情報

| 項目     | 内容                                                  |
| ------ | --------------------------------------------------- |
| システム名  | BONECD（ボネクド）                                        |
| 正式名称   | Body-Oriented Notation & Element Character Database |
| カタカナ読み | ボディ・オリエンテッド・ノーテーション・アンド・エレメント・キャラクター・データベース         |
| 和訳     | 身体指向型記法・要素キャラクターデータベース                              |
| コンセプト  | 身体情報を軸に、キャラクターの表層から深層まで全要素を体系的に記録・整理するモジュール式フレームワーク |

---

### 設計思想

BONECDは「キャラクターの情報を記録するための器」である。器は中身を選ばない。辞書が単語を載せることとその単語を肯定することが別であるように、BONECDが記録枠を提供することと記録される内容を推奨・肯定することは別である。

以下の原則に基づいて設計されている。

**身体を軸に据える** — キャラクターの存在を最も直接的に定義するのは身体である。見た目、声、仕草、体格、健康状態、性的な反応——すべて身体から始まる。内面も社会的立場も、身体を持つ存在であることを前提として成り立つ。

**モジュール式の構成を採用する** — 全項目を一枚のシートに固定するのではなく、共通ベース＋拡張パーツの構成を採用する。キャラクターによって必要な情報は異なる。必要なパーツだけを選んで使える設計が、最も多くのキャラクターと使用者に対応できる。

**ジャンルによる分類を廃止する** — キャラクターはジャンルに属さない。魔法を使うキャラクターがSNSをやっていてもいいし、現代の高校生が異能力を持っていてもいい。パーツを使うかどうかの基準は「このキャラにこの属性があるか否か」のみである。

**レーティングによるレベル分けを廃止する** — チャームポイントと性的傾向の間に優劣や段階はなく、どちらもそのキャラクターの属性の一つに過ぎない。全パーツをフラットに並べる。ゾーニングは公開時に使用者が判断すること。

**性別パーツはカテゴリ別に選択する** — 男性用・女性用でフォーマットを分けず、カテゴリ別のパーツとして提供する。どんなキャラクターでも必要なパーツだけを自由に組み合わせられる。

**フォーマット全体がプロフィールである** — 名前や年齢だけがプロフィールなのではなく、外見も性格も能力も性的傾向もすべて含めてそのキャラクターのプロフィールである。

**キャラクターシートの範疇を明確にする** — キャラクターの「状態」を記録する情報のみを対象とし、物語の展開に関する情報は対象外とする。

**内在/外在で情報の所在を区別する** — 身体・精神の中にある情報と、外との関係で生まれる情報を分ける。

**共通/拡張で情報の粒度を区別する** — 骨格レベルの基礎情報と、肉付けレベルの詳細情報を分ける。

**二つの使われ方を想定する** — 作品に必要な情報だけ効率的に管理する制作ツールとしての使用と、キャラクターを深掘りすること自体を楽しむ遊びとしての使用。どちらも正しく、どちらにも対応する。

---

### 「Body」の定義

BONECDにおける「Body」は人間の肉体に限定されない。身体とは、その存在が物理的・視覚的に現れる形態すべてを指す。人型、獣型、機械、霊体、不定形——形態を持つ存在であればBONECDの対象となる。

---

### 四系統大分類

BONECDは全パーツを二つの軸で四系統に分類する。

**第一軸：内在 / 外在**

|分類|定義|問い|
|---|---|---|
|内在|そのキャラクターが肉体的・精神的に保持しているもの。キャラクター単体で成立する属性|このキャラクターの身体や精神の中にあるか？|
|外在|外から観測されるもの、外との関係によって成立するもの。他者の目、社会的文脈、外的環境との接点で初めて意味を持つ属性|他者や環境との関係の中で現れるか？|

**第二軸：共通 / 拡張**

|分類|定義|問い|
|---|---|---|
|共通|そのキャラクターの存在を記録するために最低限必要な骨格情報。これがなければキャラクターの輪郭が掴めない|このパーツがないとキャラクターの基本像が成立しないか？|
|拡張|キャラクターの解像度を上げるための肉付け情報。なくてもキャラクターは成立するが、あれば深みが増す|このパーツはキャラクターをより詳細に知るためのものか？|

**四系統の構成**

|系統|パーツ数|性格|
|---|---|---|
|内在共通|8|身体・精神に内在する骨格レベルの基礎情報|
|内在拡張|27|身体・精神に内在する深掘りレベルの詳細情報|
|外在共通|13|外から観測される・外との関係で成立する骨格レベルの基礎情報|
|外在拡張|13|外から観測される・外との関係で成立する深掘りレベルの詳細情報|

---

### 範疇の定義

BONECDはキャラクターの「状態」を記録する。以下は範疇外とする。

|範疇外|理由|適切な管理場所|
|---|---|---|
|成長・変化|プロットの領域|脚本・プロット資料|
|死・退場|プロットの領域|脚本・プロット資料|
|関係性マトリクス|作品全体の設計|作品設計書|
|マルチフォーム|状態別にシートを分ければ済む|複数枚運用で対応|

### ゾーニングに関する注記

BONECDはキャラクター情報の記録フレームワークであり、記録された情報の公開・共有に関する判断は使用者の責任に委ねる。作品を公開する際のレーティング表示、閲覧制限、ゾーニングなどは、各プラットフォームの規約および使用者自身の判断に基づいて行うこと。
