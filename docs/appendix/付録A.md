## 付録A：FAQ
### BONECDの基本

|No.|質問|回答|
|---|---|---|
|1|BONECDとは何ですか？|身体情報を軸に、キャラクターの全要素を体系的に記録・整理するモジュール式フレームワークです。|
|2|読み方は？|「ボネクド」と読みます。|
|3|何の略ですか？|Body-Oriented Notation & Element Character Database の略です。|
|4|誰でも使えますか？|はい。創作ジャンル、経験レベル、用途を問わず誰でも使用できます。|
|5|無料ですか？|はい。ライセンスはCreative Commonsで、改変・再配布も自由です。|
|6|どんなキャラクターに使えますか？|人型、獣型、機械、霊体、不定形——形態を持つ存在であればすべて対象です。|
|7|TRPGのキャラシートとして使えますか？|使えます。共通ベース＋戦闘・能力＋魔法パーツなどを組み合わせてください。|
|8|AIにキャラクターを伝えるために使えますか？|使えます。構造化された情報なのでAIとの連携に適しています。|

### 設計思想

|No.|質問|回答|
|---|---|---|
|9|なぜ身体が軸なのですか？|キャラクターの存在を最も直接的に定義するのが身体だからです。内面も社会的立場も、身体を持つ存在であることを前提として成り立ちます。|
|10|なぜジャンル分類がないのですか？|キャラクターはジャンルに属さないからです。魔法を使うキャラがSNSをやっていてもいい。パーツ使用の基準は「このキャラにこの属性があるか否か」のみです。|
|11|なぜレーティング（レベル分け）がないのですか？|チャームポイントと性的傾向の間に優劣や段階はなく、どちらもキャラクターの属性の一つに過ぎないからです。ゾーニングは公開時に使用者が判断することです。|
|12|なぜ男性用・女性用で分かれていないのですか？|性別で二分すると、中性的・ノンバイナリー・メイクをする男性・筋肉質な女性などを排除してしまうからです。カテゴリ別にすることで自由に組み合わせられます。|
|13|BONECDは性的な内容を推奨しているのですか？|いいえ。BONECDは記録の器であり、記録枠を提供することと内容を推奨することは別です。性的パーツは必要なキャラクターにだけ使用してください。|
|14|キャラクターの成長や変化は記録できないのですか？|BONECDは「状態の記録」です。成長・変化はプロットの領域なので、脚本・プロット資料で管理してください。変化後の状態を記録したい場合は、シートを別に作成してください。|
|15|関係性マトリクスはなぜ範疇外なのですか？|関係性は作品全体の設計であり、個別キャラクターの状態ではないからです。作品設計書で管理してください。|

### 構造・分類

|No.|質問|回答|
|---|---|---|
|16|内在と外在の違いは何ですか？|内在はキャラクター単体で成立する属性（身体・精神の中にあるもの）。外在は他者や環境との関係で初めて意味を持つ属性です。|
|17|共通と拡張の違いは何ですか？|共通はキャラクターの輪郭を成立させるために最低限必要な骨格情報。拡張はなくても成立するが、あれば解像度が上がる肉付け情報です。|
|18|なぜ同じ「声」が内在と外在に分かれているのですか？|内在の「声・発声の身体性」は声を生む身体器官の動きを扱い、外在の「声・言語」は外に聞こえる声質や話し方を扱います。同じ領域でも「身体の中にある状態」と「外に現れた結果」は異なります。|
|19|全61パーツすべて使わないといけませんか？|いいえ。共通ベース21パーツが骨格で、拡張40パーツは必要なものだけ選んで使います。|
|20|パーツの並び順には意味がありますか？|あります。内在が先、外在が後。共通が先、拡張が後。キャラクターの内側から外側へ、骨格から肉付けへという順序です。|

### 使い方

|No.|質問|回答|
|---|---|---|
|21|最低限どのパーツを使えばいいですか？|最小構成は外在共通の「基本情報」「外見」＋内在共通の「性格・内面」の3パーツです。|
|22|空欄があってもいいですか？|はい。空欄は「未定義」であって「不備」ではありません。|
|23|後から項目を追加・変更してもいいですか？|はい。BONECDはスナップショットであり、いつでも更新できます。|
|24|記入例と違う書き方をしてもいいですか？|はい。すべての項目は自由記述です。記入例はあくまで参考です。|
|25|矛盾した記入をしてしまったらどうすればいいですか？|矛盾自体がキャラクターの発見になることがあります。修正しても、そのまま残しても構いません。|
|26|一つの項目に長文を書いてもいいですか？|はい。量の制約はありません。一言でも長文でも、キャラクターの情報量に応じて自由に記入してください。|
|27|拡張パーツはどこに差し込めばいいですか？|セクション3の「差し込み位置の原則」に全パーツの対応表があります。基本的に、関連する共通ベースパーツの直後に差し込みます。|
|28|同じキャラクターで複数のシートを作ってもいいですか？|はい。変身前後、時系列の違い、IFルートなど、状態別にシートを分けて管理できます。|
|29|自分で項目を追加してもいいですか？|はい。BONECDは器です。既存の項目でカバーできないものがあれば、自由に追加してください。|
|30|チームで使う場合のコツはありますか？|管理情報パーツを厳密に運用し、バージョンと担当者を明記してください。また、チーム内で「どこまでのパーツを使うか」を事前に決めておくとスムーズです。|

### 性別・性的傾向

|No.|質問|回答|
|---|---|---|
|31|性別の項目が6つもあるのはなぜですか？|現実の性別は単一の軸では表現できない多層的な概念だからです。生物学的性別・性自認・性表現・社会的性別・性的指向・恋愛的指向の6層に分解することで、あらゆる性のあり方を記録できます。|
|32|性別の6項目すべて埋める必要がありますか？|いいえ。シンプルに「男性」とだけ書きたければ生物学的性別だけ埋めれば十分です。多層的な設定が必要なキャラクターにだけ活用してください。|
|33|ロボットや無機物のキャラクターの性別はどう記入しますか？|生物学的性別を「無性」、性自認を「無性」など、該当する選択肢を記入するか、「該当なし」と記入してください。|
|34|性的傾向パーツを使わないキャラクターがいてもいいですか？|はい。内在共通の「基本的な性的傾向」は共通ベースですが、全項目を埋める義務はありません。性的な設定が不要なキャラクターは空欄で構いません。|
|35|未成年のキャラクターに性的パーツを使っていいですか？|BONECDは記録の器であり、使用を制限しません。ただし、作品を公開する際のゾーニングは使用者の責任です。各プラットフォームの規約に従ってください。|

### トラブルシューティング

|No.|質問|回答|
|---|---|---|
|40|パーツが多すぎて何から手をつけていいかわかりません|まず共通ベースの「基本情報」「外見」「性格・内面」の3パーツだけ埋めてみてください。そこから必要に応じて広げていけば大丈夫です。|
|41|内在か外在かの判断に迷います|「このキャラクター一人だけで成立する情報か？」と問いかけてください。一人で成立するなら内在、他者や環境がないと意味を持たないなら外在です。|
|42|既存のキャラクターに適用したいのですが、全部埋めるのが大変です|段階的に埋めてください。最初は既に決まっている情報だけ転記し、空欄は後から必要に応じて埋めれば十分です。|
|43|複数の作品で同じキャラクターを使い回す場合はどうすればいいですか？|作品ごとにシートを分けるか、一枚のシートに作品別の差分をメモとして追記してください。管理情報パーツの作品名で区別できます。|
|44|BONECDのパーツに載っていない情報を記録したい場合は？|自由に項目を追加してください。BONECDは器であり、拡張を制限しません。|
